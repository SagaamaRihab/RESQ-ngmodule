Emergency Evacuation Support System

RESQ is an intelligent evacuation support system designed to calculate safe and optimized routes within complex buildings, such as universities, in the event of an emergency.

Building Modeling

RESQ represents a university building as a graph, where classrooms, laboratories, staircases, and emergency exits are modeled as interconnected nodes.

Corridors and passageways between areas are represented as edges, each assigned a weight that represents the distance or traversal cost.

Evacuation Route Calculation

In the event of an emergency, RESQ automatically calculates the safest evacuation route toward one of the available exits, avoiding blocked corridors and selecting the optimal path.

The system is designed to dynamically adapt to environmental changes, providing alternative routes when necessary.

How does RESQ work?

  • Building Modeling

    The university building is modeled as a structured environment composed of classrooms, corridors, staircases and emergency exits. Each room is represented as a node, while corridors represent connections between nodes. This abstraction allows the system to digitally reproduce the physical structure of the building.

  • Evacuation Graph Construction

    The building model is converted into a graph structure used by path-finding algorithms. Nodes represent rooms and exits, while edges represent corridors with optional weights such as distance, congestion level or risk. This graph-based representation is the core of the evacuation logic.

  • Emergency Conditions Handling

    During an emergency, some corridors or areas may become inaccessible due to hazards such as fire or smoke. The system dynamically updates the evacuation graph by disabling blocked paths, ensuring that unsafe routes are automatically excluded from the computation.
